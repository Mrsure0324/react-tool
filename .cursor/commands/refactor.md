# 重构优化

请分析以下代码并提出重构方案：

{{selection}}

如果没有选中代码，则分析当前打开的文件。

## 分析步骤

### 1. 识别问题
逐项检查：
- 文件是否过长（组件 > 200 行、hook > 100 行）
- 是否有重复代码
- 职责是否单一
- 嵌套是否过深
- 命名是否清晰
- 是否有过时的写法

### 2. 输出重构方案

```
## 重构方案

### 当前问题
1. ...
2. ...

### 重构计划
| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | ... | ... |
| 2 | ... | ... |

### 影响范围
- 受影响的文件：...
- 是否有破坏性变更：是/否
```

### 3. 等待确认
方案确认后再执行。**不要直接动手改代码。**

### 4. 执行重构
- 一步一步执行，每步修改后确认无 lint 错误
- 保持功能不变（纯重构，不加新功能）
- 完成后输出改动总结
